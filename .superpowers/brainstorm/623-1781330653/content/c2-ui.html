UI / 用户体验变化

改动点对比：红色 = 移除，绿色 = 新增

原来（有冻结）

用户点「确定」
↓
[完全无响应]
dbManager.exec(原始SQL)
冻结 10-30+ 秒
↓
进度条出现
"已写入 5000/50000 行"
...
完成 ✅

改进后（无冻结）

用户点「确定」
↓
按钮: ⏳ 正在导出数据... ✓
↓
分批查询 + 写入
进度实时更新
↓
进度条
"已写入 5000 行"
...
"已写入 10000 行"
...
完成 ✅

具体 UI 改动

1. 进度条文本

当前: "已写入 12500/50000 行"（写入阶段）

改为: "已写入 12500 行"（查询+写入全程）

由于不预先 COUNT，进度条只显示绝对行数，无百分比。

2. 按钮反馈

用户点击「确定」后，按钮立即变为：

  • "⏳ 正在导出数据..."（禁用状态 + loading class）
  • 全程保持此状态，直到完成或出错

3. 完成/错误提示

  • 完成 → "已将 N 行结果写入新工作表「名称」" ✅
  • 查询出错 → "导出失败: [具体错误]" ❌
  • 写入中断 → "写入失败: [具体错误]" 保留已写入部分 ❌

涉及的 HTML 元素（全部已存在）

  • #writeSheetBtn — 导出按钮，显示 loading 状态
  • #writeProgress — 进度容器
  • #writeProgressFill — 进度条（宽度设为不可用）
  • #writeProgressText — 进度文字 "已写入 N 行"
  • #queryStatus — 最终状态消息
