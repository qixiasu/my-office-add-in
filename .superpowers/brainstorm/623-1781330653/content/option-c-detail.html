方案 C：加载状态反馈 — 设计方向

在「确定」之后立即显示反馈，但 UI 仍然会冻结

好处 👍

  • 改动极小，只需在当前 writeResultToSheet 中加几行
  • 用户至少知道系统在干活，不会以为按钮坏了
  • 保持现有架构不动

局限 ⚠️

  • UI 仍然会冻结 — sql.js 同步执行查询期间，浏览器主线程被占满
  • 如果数据量大（几十万行），浏览器可能弹出「页面无响应」提示
  • 无法显示进度，只有「等待中」
  • 长时间冻结可能让用户误以为崩溃了

你想怎么处理？

C1

就方案 C，接受冻结

加载提示 + 冻结，反正大查询不常见。只要提示够清楚就行。

C2

方案 C + 分页流式（混合）

用 LIMIT/OFFSET 分批查询 + 逐批写入 Excel，每批 5000 行。每批之间 yield 给浏览器更新进度，真正解决冻结问题。改动中等，保持架构简单。

C3

方案 B — Web Worker

完全异步化，架构最干净但改动最大。
