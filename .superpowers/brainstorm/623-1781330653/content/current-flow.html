当前导出流程 — 问题在哪里？

用户点击「确定」后 UI 完全冻结

当前：写入新工作表
1 用户输入表名，点击「确定」
⏳ dbManager.exec(原始SQL) — 全量查询
↓ sql.js 是同步执行，阻塞主线程
↓ UI 完全无响应，用户以为卡死了
2 分块写入 Excel（有进度条）

优化的核心目标

消除「确定」→「进度条出现」之间的无反馈空白期，让用户知道系统正在工作。

A

分页流式查询

将全量查询拆分为多个 LIMIT/OFFSET 子查询，分批获取数据并逐批写入 Excel，每批之间更新进度。

B

Web Worker 异步化

将 sql.js 迁移到 Web Worker 中运行，主线程永不阻塞，配合消息传递实现完整的异步 UI。

C

加载状态反馈（轻量方案）

仅在全量查询前显示一个模态加载提示「正在查询全部数据...」，保持当前架构不动。
