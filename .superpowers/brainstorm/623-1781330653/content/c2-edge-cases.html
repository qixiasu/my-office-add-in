错误处理 & 边界情况

1. 查询阶段出错

某一批的 SQL 查询返回错误（如 SQL 语法问题）。

处理：立即停止流程，恢复按钮，在 queryStatus 显示 "导出失败: [错误信息]" ❌

2. Excel 写入阶段出错

某一批 Excel.Run 失败（如工作表被删除、权限问题）。

处理：立即停止流程，恢复按钮，显示 "写入失败: [错误信息]" ❌
⚠️ 此时已写入部分数据保留，不做回滚。

3. 某批返回行数 < 每批大小

这是正常终止信号——说明已没有更多数据。

处理：正常完成流程，显示 "已将 N 行结果写入新工作表" ✅

4. 用户原始 SQL 不含 SELECT

INSERT/UPDATE/CREATE 等语句不可能进入导出流程，因为 currentQueryResult 需要 SELECT 结果。

处理：无需特殊处理——已有 if (!currentQueryResult) return; 守卫。

5. SQL 包裹子查询的兼容性

SELECT * FROM (原始SQL) LIMIT 5000 OFFSET ? 这种写法是否兼容所有 SQL？

SQLite 完全支持子查询 + LIMIT/OFFSET。
已验证兼容: JOIN / GROUP BY / ORDER BY / UNION / WHERE / HAVING / 聚合函数
⚠️ 唯一不兼容：原始 SQL 末尾带 ";" 或注释——已通过 strip 处理

6. 非预览模式（已有全量数据在内存中）

如果用户手动写了 LIMIT，或者结果本来就不足 200 行，currentQueryResult 已包含全部数据。

处理：走原有路径，不经过分页查询。保持现有逐批写入 Excel 的逻辑不变。
