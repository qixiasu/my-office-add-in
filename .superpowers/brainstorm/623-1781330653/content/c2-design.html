方案 C2 的技术设计

分页流式查询 + 逐批写入 Excel，避免 UI 冻结

改进后流程
1️⃣ 用户输入表名，点击「确定」
2️⃣ 按钮变为 ⏳ 正在查询第 1/10 批...
3️⃣ 执行 SELECT ... LIMIT 5000 OFFSET 0 → 瞬间返回
4️⃣ 写入 Excel（第 1 批），更新进度 → 已写入 5000/50000 行
5️⃣ 执行 SELECT ... LIMIT 5000 OFFSET 5000 → 瞬间返回
6️⃣ 写入 Excel（第 2 批），更新进度
⋯ 重复直到完成

关键设计问题

分页查询语法

原始 SQL 可能是任意复杂查询（JOIN、GROUP BY、ORDER BY），需要包裹为子查询：

SELECT * FROM (原始SQL) LIMIT 5000 OFFSET ?

SQLite 支持这种嵌套，但总扫描量 = 全量数据量。

行的总量？

你处理的数据集大概多少行？几十万？几万？这影响分批策略。
