# Concatenate Columns - 功能设计

## 概述

用户选中 Excel 中的任意两列，点击功能区按钮，在第二列后面插入一列，将这两列的数据用指定连接符连接起来。

## 用户交互流程

1. 用户在 Excel 中选中两列（可连续或不连续）
2. 点击功能区按钮（如 "连接列"）
3. 弹出输入框让用户指定连接符（默认 `_`）
4. 程序在第二列右侧插入新列
5. 新列写入连接后的纯文本值

## 数据处理规则

| 场景 | 处理方式 |
|------|----------|
| 单元格有值 | 直接连接：`值1 + 连接符 + 值2` |
| 单元格为空 | 保留连接符：`"" + "_" + "hello"` → `"_hello"` |
| 某行无数据 | 整行视为空，跳过 |

## 实现方案

### 核心逻辑

1. 获取用户选中的区域
2. 验证至少选中两列
3. 弹出对话框获取连接符
4. 读取两列的单元格值
5. 在第二列右侧插入一列
6. 将连接结果作为纯文本写入新列

### Excel API 使用

- `context.workbook.getSelectedRange()` 获取选中区域
- `range.getColumnCount()` 验证列数
- `range.getAbsoluteResizedRange()` 获取完整区域
- `context.workbook.worksheets.getItem()` 插入新列
- `range.getCell()` 写入值

### 文件结构

- `src/commands/commands.js` - 功能区按钮入口
- `src/taskpane/taskpane.js` - 任务窗格（暂无变化）

## 错误处理

| 错误 | 处理 |
|------|------|
| 选中少于两列 | 提示用户"请至少选择两列" |
| 用户取消输入 | 不执行任何操作 |
| 空工作表 | 静默处理，不插入列 |

## 测试场景

| 场景 | 输入 | 连接符 | 预期结果 |
|------|------|--------|----------|
| 标准连接 | A1="hello", B1="world" | `_` | C1="hello_world" |
| 包含空值 | A1="", B1="hello" | `_` | C1="_hello" |
| 数字连接 | A1=123, B1=456 | `-` | C1="123-456" |
| 多行 | A1:A3 有值 | `_` | C1:C3 对应连接结果 |
