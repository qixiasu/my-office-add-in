# Count-Values 静态值替代公式设计方案

**日期:** 2026-05-29
**状态:** 已批准

## 背景

当前实现使用 `COUNTIF` 公式填充，当选中列的行数特别多时，Excel 会因为公式重算而卡顿。需要改为在 JavaScript 中直接计算计数，然后写入静态值。

## 设计

### 核心改动

**数据流程：**
1. 读取选中列的数据到 JS 内存
2. 用 `Map` 统计每个值的出现次数
3. 直接将计数结果作为静态数字写入新列（不使用公式）

### 步骤详解

**1. 读取数据**
保持现有的 `usedRange.values` 读取方式，将 2D 数组展平成 1D 数组遍历。

**2. 计数（Map）**
```javascript
const countMap = new Map();
values.forEach(row => {
  const key = row[0]; // 原始值
  countMap.set(key, (countMap.get(key) || 0) + 1);
});
```

**3. 写入静态值**
对数据区域每一行，查找该行的值在 Map 中的计数，直接写入：
```javascript
resultCell.values = [[countMap.get(cellValue)]];
```

**4. 保留的功能**
- 插入新列逻辑不变
- 表头命名逻辑不变
- 多列处理从右到左的顺序不变

**5. 错误处理**
- 空值处理：Map 中不存在的 key 视为 0 次
- 读取失败时回退错误提示

### 性能说明

Excel 行数限制约 100 万，JS 的 Map 处理这个量级毫秒级，内存占用合理（每行一个短字符串约 2-10 字节）。

### 限制

- 始终写入静态值，不使用公式
- 完全匹配计数，不支持模糊匹配
