# 增强查找 — 设计文档

**日期**: 2026-05-24
**状态**: 待实施

## 问题

传统 VLOOKUP 有两个痛点：

1. **多列查找繁琐**：需要频繁 VLOOKUP 多列数据时，要为每列逐个编写/修改公式，操作重复且容易出错
2. **索引列必须在左侧**：VLOOKUP 要求匹配列必须在返回列的左侧，当索引列在右侧时，用户需要先手动复制或剪切列到前面，破坏原有数据结构

## 方案

使用 INDEX/MATCH 公式替代 VLOOKUP，通过弹窗配置界面，一次配置完成多列查找。

**选择 INDEX/MATCH 的理由**：
- 天然支持匹配列在任意位置（左、右均不限）
- 兼容性好，Excel 2007+ 都支持
- 一次 MATCH 结果可被多个 INDEX 复用，多列查找效率更高

### 交互逻辑

1. **查找表区域**：默认填入用户当前选中区域，也可手动编辑或点击"选择区域"按钮在 Excel 中点选
2. **表头行**：默认第 1 行，修改后重新读取表头更新勾选列表
3. **匹配列**：下拉显示查找表的所有列（A、B、C...），用户选择用哪一列作为匹配键
4. **返回列**：勾选列表显示列号+表头文字（如 "B - 部门"），支持全选/取消全选
5. **数据预览**：实时显示查找表前几行数据，帮助确认配置正确

### 处理流程

**公式模式**：
1. 弹窗收集配置参数，通过 `messageParent()` 发给父页面
2. 父页面在 `Excel.run` 中遍历数据区域每一行
3. 为每个勾选的返回列生成 INDEX/MATCH 公式并写入
4. 使用 autoFill 自动填充剩余行

**静态值模式**：
1. 弹窗内 `Excel.run` 读取查找表全部数据到内存
2. JS 中做本地哈希匹配（O(n+m)）
3. 将匹配结果传回父页面
4. 父页面在 `Excel.run` 中一次性写入结果数组

### 公式生成规则

```
数据区域 A1:A100（待查找的值）
查找表 Sheet2!A1:D500（匹配列 A，勾选返回 B 和 D）

数据行1 → B列公式: =INDEX(Sheet2!$B$1:$B$500, MATCH(A1, Sheet2!$A$1:$A$500, 0))
数据行1 → D列公式: =INDEX(Sheet2!$D$1:$D$500, MATCH(A1, Sheet2!$A$1:$A$500, 0))
```

- 查找表引用使用绝对行号范围（`$B$1:$B$500`），确保跨工作表稳定
- MATCH 的 lookup_value 使用相对引用（`A1`），自动适配每行
- 匹配模式：0 为精确匹配，1 为近似匹配

### 输入验证

| 场景 | 处理 |
|------|------|
| 未选择数据区域 | 提示"请先在 Excel 中选择要查找的数据区域" |
| 查找表区域为空或无效 | 提示"请输入有效的查找表区域" |
| 表头行超出查找表范围 | 提示"表头行号超出查找表行数" |
| 未勾选任何返回列 | 提示"请至少勾选一列作为返回列" |
| 数据量超过 1050000 行 | 提示"数据量过大（N 行），单次最多支持 1050000 行" |

### 边界情况

| 场景 | 处理 |
|------|------|
| 查找表中无匹配值 | 显示 `#N/A` |
| 查找表有空值 | 正常匹配，空值按空字符串处理 |
| 合并单元格 | null 值视为空值 |
| 公式单元格 | 使用计算结果值 |
| 跨工作簿 | 用户手动输入含工作簿名的区域地址 |
