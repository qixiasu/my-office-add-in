# 跨文件 Sheet 合并功能设计

## 功能概述

选择多个外部 Excel 文件，读取每个文件的当前激活 sheet，按照统一表头行号设置进行合并，结果写入当前工作簿的新 sheet，并添加来源文件名标识列。

## 用户场景

- 少量文件（2-10 个）：快速选择并合并
- 大量文件（10 个以上）：稳定完成并有进度提示

### 组件说明

| 组件 | 说明 |
|------|------|
| 文件选择按钮 | 触发 `<input type="file" multiple accept=".xlsx">` |
| 文件列表 | 显示已选文件，支持逐个移除 |
| 表头行号输入 | 数字输入，统一应用于所有文件 |
| 执行按钮 | 触发合并操作 |
| 状态提示 | 显示处理进度和结果 |

## 技术方案

| 组件 | 技术 |
|------|------|
| 文件选择 | HTML `<input type="file" multiple accept=".xlsx">` |
| Excel 解析 | SheetJS (xlsx) 库（浏览器端解析） |
| 数据写入 | Office JavaScript API |

### 库引入

通过 CDN 或本地引入 SheetJS：
```html
<script src="https://cdn.sheetjs.com/xlsx-0.20.1/package/dist/xlsx.full.min.js"></script>
```

## 数据流

```
用户选择文件（多选 .xlsx）
     ↓
SheetJS 解析每个文件的当前激活 sheet
     ↓
统一表头处理（第一文件取表头，其他跳过表头行）
     ↓
新增"来源文件"列为第一列
     ↓
写入当前工作簿新 sheet（如"合并结果"）
     ↓
弹出成功提示框
```

## 合并规则

| 规则 | 说明 |
|------|------|
| 表头行号 = 0 | 无表头，所有行都是数据 |
| 表头行号 >= 1 | 对应行作为表头，后续为数据 |
| 列数不一致 | 中止合并，报错提示 |
| 新 sheet 已存在 | 自动重命名（如"合并结果_1"） |
| 空文件 | 跳过该文件并提示警告 |
| 合并顺序 | 按用户选择文件的顺序依次合并 |

## 表头与数据列处理

1. **第一文件**：读取表头行 + 所有数据行
2. **后续文件**：跳过表头行，读取所有数据行
3. **结果表头**：仅保留第一文件的表头（如有）
4. **来源标识**：新增"来源文件"列为第一列，值为对应文件名

## 文件名标识列

- **位置**：最左列（第一列）
- **列名**：无表头时使用 "来源文件" 作为列名
- **内容**：对应原始 Excel 文件的文件名（含扩展名）

## 成功提示框

```
✅ 合并成功！
共合并 N 个文件，M 行数据（Z 列）
输出工作表：合并结果
```

提示框使用自定义 DOM 模态框（项目已有 `office-addin-no-native-dialogs` 规范限制）。

## 错误处理

| 错误场景 | 处理方式 |
|----------|----------|
| 列数不一致 | 弹出错误提示，中止合并 |
| 文件解析失败 | 跳过该文件，继续处理其他文件 |
| Sheet 不存在 | 跳过该文件并提示警告 |
| 空数据 | 跳过该文件并提示警告 |
| Office API 错误 | 显示错误信息到状态区域 |

## 进度反馈

对于大量文件（10+），在状态区域显示：
- "正在处理 file1.xlsx..."
- "已处理 3/N 个文件..."
- 最终完成时显示汇总信息
