# 增强查找 - lookup_value 参数界面设计

## 背景

VLOOKUP 函数结构：
- lookup_value — 要查找的值（一列）
- table_array — 查找表区域

当前问题：用户选中区域后自动取第一列作为 lookup_value，行为隐式，用户无法配置。

## 功能需求

1. 用户可重新指定 lookup_value 来源（当前选中区域的第一列）
2. 支持同一 sheet、不同 sheet、不同工作簿的场景
3. 查找值区域输入框不可编辑，只展示 + 刷新选择

## UI 设计

### 刷新目标切换

弹窗顶部新增切换控件（类似选项卡）：

```
[ 刷新查找值 ] [ 刷新查找表 ]
```

- 当前激活项高亮显示
- 点击切换刷新目标
- "刷新选择"按钮根据当前激活项，更新对应字段

### 查找值区域（新增字段）

```
标签：查找值区域：
输入框：[只读，不可编辑] [刷新选择按钮]
说明：自动取自当前选区第一列
```

- 输入框显示当前选中的区域地址（如 Sheet1!A2:A100）
- 不可手动编辑，防止输入错误格式
- "刷新选择"在"刷新查找值"模式下工作

### 查找表区域（现有字段）

逻辑保持不变。

## 数据流

1. 用户选中区域 → 打开弹窗 → 第一列地址自动填入"查找值区域"
2. 用户手动填写"查找表区域"（或刷新选择）
3. 用户配置匹配列、返回列等
4. 执行时读取"查找值区域"第一列作为 lookup_value 数组

## 状态持久化

刷新切换状态也需要通过 URL 参数传递，以便下次打开时恢复。

## 执行流程变化

原有流程：
用户选中区域 → 打开弹窗 → 自动取第一列作为 lookup_value

新流程：
用户选中区域 → 打开弹窗 →
  - 查找值区域 = 选中区域第一列（只读）
  - 查找表区域 = 用户手动填写
  → 用户可随时刷新选择切换目标，重新选择区域

## 涉及文件

- src/commands/vlookup-dialog.html — 对话框 UI 修改
- src/commands/commands.html — 刷新选择逻辑调整（区分两种刷新目标）

## 交互状态

刷新目标激活项 | 刷新选择结果 | 更新字段
---|---|---
刷新查找值 | 用户重新框选区域 | 查找值区域输入框
刷新查找表 | 用户重新框选区域 | 查找表区域输入框
