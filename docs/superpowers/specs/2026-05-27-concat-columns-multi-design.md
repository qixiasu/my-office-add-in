# 连接列功能扩展设计

**日期**: 2026-05-27
**状态**: 已批准
**需求**: 将连接列功能从仅支持2列扩展为支持任意多列（用户选择多少列就连接多少列），超过3列时执行前弹出确认框。

---

## 1. 功能行为

- 用户选中 N 列（N ≥ 2）
- 输入连接符（默认 `_`）
- 点击「执行连接」
- **若 N ≤ 3**：直接执行，在所选范围**最右侧插入新列**，写入拼接公式
- **若 N > 3**：弹出确认框，显示「将连接第 X 列到第 Y 列，使用连接符【Z】」，用户确认后执行；用户取消则不执行

### 1.1 结果列位置

插入到所选范围的最右侧（原2列逻辑的自然扩展）。

---

## 2. 公式构造

沿用 `&` 拼接方式，兼容所有 Excel 版本（IE11+）。

### 2.1 公式模式

```
=IF(A1&B1&C1="","",A1&"_"&B1&"_"&C1)
```

### 2.2 生成规则

1. 遍历所选每列，依次拼入 `A1&B1&C1...`（用于 IF 空值判断）
2. 连接符插在每对相邻项之间：`&"连接符"&`
3. 最外层 IF 判断整行是否全空，全空则输出空字符串

### 2.3 示例对照

| 列数 | 公式 |
|------|------|
| 2 | `=IF(A1&B1="","",A1&"_"&B1)` |
| 3 | `=IF(A1&B1&C1="","",A1&"_"&B1&"_"&C1)` |
| 4 | `=IF(A1&B1&C1&D1="","",A1&"_"&B1&"_"&C1&"_"&D1)` |

---

## 3. 确认对话框

使用原生 `window.confirm()`。

### 3.1 显示内容

- 消息格式：`"将连接第 {起始列} 列到第 {终止列} 列，使用连接符【{连接符}】"`
- 示例：`"将连接第 A 列到第 D 列，使用连接符【_】"`

### 3.2 交互

- 用户点「确定」→ 执行插入逻辑
- 用户点「取消」→ 关闭确认框，不执行，按钮恢复可用，状态保持「等待操作...」

## 5. 核心函数设计

### 5.1 `buildNConcatFormula(columns, connector)`

**参数**:
- `columns`: 字符串数组，如 `['A', 'B', 'C']`
- `connector`: 连接符字符串，如 `'_'`

**返回值**: 公式字符串

**逻辑**:
1. 生成空值判断字符串：`columns.map(c => c + "1").join('&')`
2. 生成拼接字符串：`columns.map((c, i) => i === 0 ? c + '1' : '"连接符"' + '"&' + c + '1').join('&')`
3. 返回外层 IF 包装

## 6. 测试场景

| 场景 | 输入 | 预期行为 |
|------|------|---------|
| 2列直接执行 | 选 A、B 列 | 无确认框，直接执行 |
| 3列直接执行 | 选 A、B、C 列 | 无确认框，直接执行 |
| 4列需确认 | 选 A、B、C、D 列 | 弹出确认框，确认后执行，取消则不执行 |
| 单列报错 | 选 A 列 | 报错「请至少选择两列」 |
| 空值处理 | A1=空, B1="x" | IF 判断输出空字符串 |
| 连接符含双引号 | 连接符为 `a"b` | 转义为 `a""b` 后拼入公式 |
| 行数超限 | 超过 MAX_ROWS | 报错并阻止执行 |

---

## 7. 错误处理

- **列数 < 2**：`"错误: 请至少选择两列"`
- **行数超限**：超过 MAX_ROWS (1050000) 行，报错并阻止执行
- **Excel API 错误**：捕获并显示 `"错误: " + error.message`
