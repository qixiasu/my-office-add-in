# 填充序列工具 - 设计文档

## 一、概述

**功能描述：** 用户在 Excel 中选中一个区域，通过侧边栏面板输入起始值和步长，对选中区域按线性序列填充数值，支持整数和小数（包括负数）。

**用户场景：** 快速生成序号、序列数据、阶梯数值等。

## 三、UI 设计

### 布局（水平布局）

```
┌─────────────────────────────────┐
│  填充序列工具                       │
│                                 │
│  [说明卡片：选中区域按起始值和步长填充]  │
│                                 │

│  [        执行填充        ]       │
│                                 │
│  状态：等待操作...                │
└─────────────────────────────────┘
```

### 详细规格

- **布局：** 水平布局（起始值和步长左右并排，各 50% 宽度）
- **输入框：** 自动聚焦（autoFocus）
- **按钮：** 下方居中，100% 宽度
- **状态消息：** 按钮下方

### 输入验证规则

| 字段 | 规则 |
|------|------|
| 起始值 | 必填，支持整数和小数，支持负数（如 `-1`, `0.5`, `-0.25`） |
| 步长 | 必填，支持整数和小数，支持负数（如 `1`, `-0.5`） |

数值格式：`/^-?\d+(\.\d+)?$/`（可选负号，整数或小数）

## 四、执行逻辑

### 流程

1. 用户点击"执行填充"按钮
2. 读取 Excel 当前选中区域（`Excel.run` + `getSelectedRange`）
3. 获取区域行数和列数（`rowCount`, `columnCount`）
4. 计算总单元格数：`total = rowCount × columnCount`
5. 生成 1D 序列数组：`[start, start+step, start+2*step, ...]`，共 `total` 个
6. 将 1D 数组转换为 2D 数组（列优先，逐列填充）
7. 将 2D 数组写入选中区域（`range.values = values`）
8. 显示成功状态："完成! 已填充 N 个单元格"

### 填充示例

```
选中区域：2行 × 3列
起始值 = 1，步长 = 10

生成数组：[1, 11, 21, 31, 41, 51]
填充结果：
  A列  B列  C列
  1    11   21
  31   41   51
```

## 五、错误处理

| 场景 | 状态消息 |
|------|----------|
| 用户未选择区域 | "错误: 请先在 Excel 中选择单元格区域" |
| 起始值为空 | "错误: 请输入起始值" |
| 步长为空 | "错误: 请输入步长" |
| 数值格式无效 | "错误: 请输入有效的数字" |
| 数据量过大（> 1050000） | "错误: 数据量过大（{n} 个单元格），单次最多支持 1050000 个" |

## 六、状态消息样式

| 状态 | 样式类 |
|------|--------|
| 空闲 | `status-message status-idle` |
| 处理中 | `status-message status-loading` |
| 成功 | `status-message status-success` |
| 错误 | `status-message status-error` |

## 七、数据流

```
用户输入（起始值、步长）
      ↓
点击"执行填充"按钮
      ↓
Excel.run → getSelectedRange
      ↓
计算 total = rowCount × columnCount
      ↓
生成 1D 序列数组
      ↓
转换为 2D 数组（列优先）
      ↓
range.values = 2D数组
      ↓
显示成功状态
```

## 八、技术约束

- 使用纯 JavaScript（与现有 taskpane 一致）
- 无需额外依赖
- 复用 `getColumnLetter` 等现有工具函数
