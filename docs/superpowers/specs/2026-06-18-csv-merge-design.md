# CSV 文件合并功能设计

## 概述

开发一个新功能，将多个 CSV 文件合并成一个新 CSV 文件。支持自动检测各文件的编码格式和分隔符，提供三种表头处理模式，用户可指定输出编码和分隔符，最终结果通过浏览器下载。

## 交互流程

1. 用户在功能区点击按钮，打开任务面板
2. 用户通过文件选择器选择多个 CSV 文件（数量不限制）
3. 用户配置表头处理模式、输出编码、输出分隔符
4. 点击"合并并下载"，浏览器自动下载合并后的 CSV 文件

## 核心处理逻辑

### 文件读取

- 每个文件独立检测编码（UTF-8 / GBK / GB2312 / Big5）和分隔符（逗号 / 制表符 / 分号 / 单竖线 `|`）
- 复用现有 `csv-utils.js` 的分块读取能力
- 用户指定的**输出编码/分隔符**仅用于最终输出文件，不影响读取

### 表头处理（三种模式）

| 模式 | 行为 |
|------|------|
| 跳过所有表头行 | 所有文件跳过第 1 行，从第 2 行开始合并 |
| 保留第一个文件的表头 | 文件 1 的第 1 行作为输出表头，文件 1 从第 2 行合并；其他文件跳过第 1 行 |
| 将所有表头行作为数据合并 | 不跳过任何行，所有内容纵向追加 |

### 列对齐

- 读取所有文件表头后，取并集作为统一表头
- 某文件缺失某列 → 该列在该文件数据行中留空
- 表头行按统一列顺序输出

### 分块写入最终文件

- 使用分块拼接避免内存爆炸：读取每个文件的一批行、处理列对齐后追加到输出缓冲
- 最终生成 Blob → 通过隐藏 `<a download>` 触发下载

## 组件拆解

| 模块 | 职责 |
|------|------|
| `csv-merge-taskpane.js/html` | UI 交互、文件选择、表头模式选择、触发合并 |
| `src/utils/csv-merge-utils.js` | 核心合并逻辑：读取多文件、列对齐、生成输出缓冲、触发下载 |
| `src/utils/csv-utils.js` | 已有 CSV 解析能力 — 复用 `parseCSVFile` 分块读取 |

## 错误处理

- 文件大小为 0 → 跳过该文件并警告
- 合并后无数据 → 提示并阻止下载
- 分块处理兜底，避免内存爆炸

## 测试要点

- 同编码/同分隔符的多个文件合并
- 不同编码文件合并（混合格并）
- 不同分隔符文件合并
- 三种表头模式的各自输出
- 列不对齐时的留空处理
- 大文件（>10MB）的合并
